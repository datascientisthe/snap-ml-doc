class snap_ml_mpi.RidgeRegression.RidgeRegression(max_iter=50, regularizer=1.0, device_ids=[], verbose=False, use_gpu=False, dual=True, num_threads=32)¶

Ridge Regression classifier

This class implements regularized ridge regression using the IBM Snap ML solver. It handles both dense and sparse matrix inputs. Use csr, csc or ndarray matrix format for training and csr or ndarray format for prediction.

Parameters:
  • max_iter (int, default : 50) – Maximum number of iterations used by the solver to converge.
  • regularizer (float, default : 1.0) – Regularization strength. It must be a positive float. Larger regularization values imply stronger regularization.
  • use_gpu (bool, default : False) – Flag for indicating the hardware platform used for training. If True, the training is performed using the GPU. If False, the training is performed using the CPU.
  • device_ids (array-like of int, default : []) – If use_gpu is True, it indicates the IDs of the GPUs used for training. For single GPU training, set device_ids to the GPU ID to be used for training, e.g., [0]. For multi-GPU training, set device_ids to a list of GPU IDs to be used for training, e.g., [0, 1].
  • dual (bool, default : True) – Dual or primal formulation. Recommendation: if n_samples > n_features use dual=True.
  • verbose (bool, default : False) – If True, it prints the training cost, one per iteration. Warning: this will increase the training time. For performance evaluation, use verbose=False.
  • num_threads (int, default : 32) – The number of threads used for running the GPU training. The value of this parameter should be a power of 2. This parameter will be ignored when running training on CPU.
Variables:

coef (array-like, shape (n_features,)) – Coefficients of the features in the trained model.

Logistic Regression classifier

This class implements distributed regularized logistic regression using the IBM Snap ML MPI solver. It handles both dense and sparse matrix inputs. Use csr, csc or ndarray matrix format for training and csr or ndarray format for prediction. We recommend the user to first normalize the input values.

For single-GPU training, set device_ids to the GPU ID to be used for training, e.g., [0]. For multi-GPU training, set device_ids to a list of GPU IDs to be used for training, e.g., [0, 1].
  • class_weight ('balanced' or None, optional) – If set to ‘None’, all classes will have weight 1.
  • dual (bool, default : True) – Dual or primal formulation. Recommendation: if n_samples > n_features use dual=True.
  • verbose (bool, default : False) – If True, it prints the training cost, one per iteration. Warning: this will increase the training time. For performance evaluation, use verbose=False.
  • num_threads (int, default : 32) – The number of threads used for running the GPU training. The value of this parameter should be a power of 2. This parameter will be ignored when running training on CPU.

Support Vector Machine classifier

This class implements regularized support vector machine using the IBM Snap ML solver. It handles both dense and sparse matrix inputs. Use csr or ndarray matrix format for both training and prediction. The training uses the dual formulation. We recommend the user to normalize the input values.

Parameters:

  • regularizer (float, default : 1.0) – Regularization strength. It must be a positive float. Larger regularization values imply stronger regularization.
  • use_gpu (bool, default : False) – Flag for indicating the hardware platform used for training. If True, the training is performed using the GPU. If False, the training is performed using the CPU.
  • device_ids (array-like of int, default : []) – If use_gpu is True, it indicates the IDs of the GPUs used for training. For single GPU training, set device_ids to the GPU ID to be used for training, e.g., [0]. For multi-GPU training, set device_ids to a list of GPU IDs to be used for training, e.g., [0, 1].
  • class_weight ('balanced' or None, optional) – If set to ‘None’, all classes will have weight 1.
  • verbose (bool, default : False) – If True, it prints the training cost, one per iteration. Warning: this will increase the traiing time. For performance evaluation, use verbose=False.
  • num_threads (int, default : 32) – The number of threads used for running the GPU training. The value of this parameter should be a power of 2. This parameter will be ignored when running training on CPU.
